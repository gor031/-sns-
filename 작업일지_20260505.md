### 1. 이미지가 캔버스 왼쪽 상단에 쳐박히는 버그 수정
**파일**: `components/studio/DesignStudio.tsx`

**원인**:
- Fabric.js v7에서 기본 `originX/originY`가 `'center'`
- `left: 0, top: 0`으로 설정하면 이미지 **중심**이 (0,0)에 위치 → 이미지가 좌상단으로 밀림

**수정 위치 3곳**:
1. `useEffect` → `initialImageUrl` 처리 시 이미지 로드
2. `addImage()` → 첫 번째 이미지 (캔버스 새로 맞춤)
3. `addImage()` → 추가 이미지 중앙 배치

**수정 내용**:
```tsx
img.set({
  left: 0,
  top: 0,
  scaleX: 1,
  scaleY: 1,
  originX: 'left',  // 추가
  originY: 'top',   // 추가
});
```

### 2. AI 객체 분리 시 이미지 품질 저하 수정
**파일**: `components/studio/DesignStudio.tsx` (`handleAIExtract` 함수)

**원인**:
- `toDataURL({ format: 'png', multiplier: 1 })` → 캔버스에 축소된 크기 그대로 저해상도 캡처
- AI에 저해상도 이미지가 전송되어 분석 품질 저하

**수정 내용**:
```tsx
// 원본 이미지 element에서 직접 원본 해상도 캡처
const imgElement = activeObj.getElement() as HTMLImageElement;
let dataURL: string;
if (imgElement && (imgElement.src || imgElement.currentSrc)) {
  const tempCanvas = document.createElement('canvas');
  tempCanvas.width = objWidth;
  tempCanvas.height = objHeight;
  const ctx = tempCanvas.getContext('2d');
  ctx?.drawImage(imgElement, 0, 0, objWidth, objHeight);
  dataURL = tempCanvas.toDataURL('image/png');
} else {
  // fallback: 스케일 역수로 원본 해상도에 근접하게 캡처
  const captureMultiplier = Math.min(4, Math.max(2, Math.round(1 / Math.min(Math.abs(objScaleX), Math.abs(objScaleY)))));
  dataURL = activeObj.toDataURL({ format: 'png', multiplier: captureMultiplier });
}
```

### 6. AI 백엔드 Grounding DINO + SAM 2로 전면 교체

**기존 방식의 문제**:
- SAM 자동 마스크: 576개 랜덤 포인트 → 뭘 분리할지 모름 → 엉망

**새로운 파이프라인**:
```
EasyOCR → 텍스트 위치/내용 인식 → 편집 가능한 텍스트 레이어
Grounding DINO → "logo, icon, photo, shape, object" 텍스트 프롬프트로 의미 단위 객체 탐지
SAM 2 → 탐지된 bbox를 box prompt로 정밀 마스크 생성
원본 픽셀 그대로 crop → 재생성 없음, 원본 화질 유지
cv2.inpaint → 배경 복원
```

**설치 라이브러리**:
- `transformers>=4.40.0` → Grounding DINO (`IDEA-Research/grounding-dino-base`)
- `git+https://github.com/facebookresearch/sam2.git` → SAM 2 (`facebook/sam2.1-hiera-small`)
- `rembg` → bg-remove 전용
- `easyocr` → 텍스트 인식

**핵심 코드**:
```python
# DINO로 객체 탐지
```
